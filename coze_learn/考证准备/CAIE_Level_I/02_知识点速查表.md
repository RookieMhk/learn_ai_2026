## 一、AI基础认知（占比约5%）

### 1.1 AI定义与发展
- **AI（人工智能）**：使机器具备人类智能的技术科学
- **关键里程碑**：
  - 1956年：达特茅斯会议，AI概念正式诞生
  - 2012年：深度学习突破，AlexNet引发AI革命
  - 2017年：Transformer架构提出
  - 2020年后：大语言模型（LLM）爆发

### 1.2 AI伦理与法规
| 伦理原则 | 说明 |
|----------|------|
| 公平性 | AI不应产生歧视性结果 |
| 透明性 | AI决策过程可解释 |
| 隐私保护 | 用户数据安全使用 |
| 安全性 | AI系统安全可控 |
| 责任归属 | 明确AI造成问题的责任主体 |

## 二、大模型核心机制（占比约10%）

### 2.1 机器学习基础
- **监督学习**：有标注数据的训练
- **无监督学习**：无标注数据中发现模式
- **强化学习**：通过奖励信号学习策略

### 2.2 深度学习核心概念
- **神经网络**：模仿人脑神经元的计算模型
- **反向传播**：训练神经网络的核心算法
- **Transformer**：当前主流的大模型架构

### 2.3 大模型关键技术
| 术语 | 含义 |
|------|------|
| **Token** | 大模型处理文本的基本单位（可以是字、词或子词） |
| **Embedding** | 将文本转化为向量的技术，使语义相似的内容距离更近 |
| **幻觉** | 大模型生成看似合理但实际错误的内容 |
| **上下文窗口** | 模型一次能处理的最大文本量 |

## 三、结构化思维与AI交互（占比约15%）【重点】

### 3.1 结构化思维四原则
1. **结论先行**：先给出结论，再提供支撑
2. **以上统下**：上层是下层的概括
3. **归类分组**：相似信息归为一组
4. **逻辑递进**：按某种逻辑顺序排列

### 3.2 5W2H框架
| 要素 | 含义 | Prompt应用 |
|------|------|------------|
| What | 做什么 | 明确任务目标 |
| Who | 谁来做 | 确定角色/对象 |
| When | 何时 | 设定时间限制 |
| Where | 何地 | 明确场景/环境 |
| Why | 为什么 | 解释目的/原因 |
| How | 怎么做 | 描述方法/步骤 |
| How much | 程度/数量 | 量化要求 |

### 3.3 SCQA框架
- **S（Situation）**：情境——描述背景
- **C（Complication）**：冲突——指出问题
- **Q（Question）**：疑问——提出核心问题
- **A（Answer）**：答案——给出解决方案

**应用示例**：用于写报告、做方案时的结构化表达

### 3.4 需求到指令的转化
```
模糊需求 → 结构化分析 → 精准Prompt
例："帮我写点东西" 
→ 明确：写什么？给谁看？要达到什么目的？
→ 精准："以营销总监的口吻，写一封200字的产品推介邮件，目标用户是25-35岁职场女性，突出环保和便捷两个卖点"
```

## 四、Prompt设计（占比约25%）【最高频考点】

### 4.1 Prompt基本要素
| 要素 | 说明 | 示例 |
|------|------|------|
| 角色 | AI扮演的身份 | "你是一位资深营销专家" |
| 任务 | 明确要做什么 | "帮我撰写产品文案" |
| 背景 | 必要的上下文 | "产品是一款智能手表" |
| 要求 | 输出格式/风格等 | "字数控制在200字以内" |
| 约束 | 限制条件 | "避免使用专业术语" |

### 4.2 进阶Prompt技巧

**Few-shot（少样本学习）**
```
示例：
输入：我很高兴 → 情感标签：正面
输入：今天下雨了 → 情感标签：中性
输入：考试没考好 → 情感标签：负面
输入：升职加薪了 → 情感标签：？
```

**CoT（Chain of Thought，思维链）**
```
要求AI展示推理过程：
"请分步骤解答，每一步说明原因。"
```

**角色设定（Role Prompting）**
```
"你是一位拥有10年经验的资深律师，擅长合同审核。"
```

### 4.3 多模态Prompt
| 类型 | 说明 |
|------|------|
| 文生图 | 根据文本描述生成图片 |
| 文生视频 | 根据文本描述生成视频 |
| 图生文 | 分析图片内容并描述 |

## 六、RAG与Agent（占比约18%）【高阶考点】

### 6.1 RAG（检索增强生成）

**工作原理**：
```
用户问题 → 检索外部知识 → 将检索结果加入Prompt → 大模型生成回答
```

**核心优势**：
- 解决大模型知识过时问题
- 减少幻觉，提升准确性
- 支持私有知识库

### 6.2 Agent（智能体）

**核心组件**：
| 组件 | 功能 |
|------|------|
| 感知（Perception） | 获取环境信息 |
| 规划（Planning） | 制定行动策略 |
| 行动（Action） | 执行具体操作 |
| 记忆（Memory） | 存储历史信息 |

**Agent能力等级**：
1. **反射型**：基于规则响应
2. **记忆型**：能记住上下文
3. **目标型**：能规划路径达成目标
4. **反思型**：能自我评估和调整

### 6.3 多智能体协作
- **分工协作**：不同Agent负责不同子任务
- **串行执行**：按顺序完成任务链
- **并行执行**：多个Agent同时处理不同任务
- **人机协作**：人类在关键节点介入审核
