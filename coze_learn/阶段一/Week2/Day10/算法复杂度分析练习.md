# 算法复杂度分析练习
## 掌握时间复杂度与空间复杂度的核心概念与分析方法

### 一、复杂度分析的重要性
在AI和算法面试中，复杂度分析是评估算法效率的关键工具。它能帮助我们：
1. **预测算法性能**：估计算法在不同规模数据下的运行时间
2. **选择最优算法**：在多个解决方案中挑选最有效的
3. **优化代码性能**：识别性能瓶颈并进行改进
4. **通过技术面试**：大厂面试必考内容

### 二、时间复杂度（Time Complexity）

#### 2.1 大O表示法（Big O Notation）
**定义**：描述算法运行时间随输入规模增长的变化趋势，关注最坏情况下的增长级别。

**核心原则**：
1. **忽略常数项**：O(2n) → O(n)
2. **忽略低阶项**：O(n² + n) → O(n²)
3. **关注最高阶项**：决定算法的主要增长趋势

#### 2.2 常见时间复杂度类别

##### O(1) - 常数时间复杂度
**原理**：算法的执行时间不随输入规模变化。

**示例**：数组按索引访问、哈希表查找
```python
def constant_time_example(arr, index):
    """O(1) - 直接访问数组元素"""
    return arr[index]  # 无论数组多大，访问时间相同

# 数学公式：T(n) = c（常数）
```

##### O(log n) - 对数时间复杂度
**原理**：每次操作将问题规模减半，常见于二分查找、平衡二叉搜索树。

**示例**：二分查找
```python
def binary_search(arr, target):
    """O(log n) - 二分查找"""
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2  # 每次减半
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

# 数学公式：T(n) = c * log₂(n)
# 推导：n → n/2 → n/4 → ... → 1，需要 log₂(n) 步
```

##### O(n) - 线性时间复杂度
**原理**：执行时间与输入规模成正比。

**示例**：遍历数组、线性搜索
```python
def linear_search(arr, target):
    """O(n) - 线性搜索"""
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

# 数学公式：T(n) = c * n
```

##### O(n log n) - 线性对数时间复杂度
**原理**：常见于高效排序算法，如归并排序、快速排序。

**示例**：归并排序
```python
def merge_sort(arr):
    """O(n log n) - 归并排序"""
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[:mid])
    
    # 合并操作 O(n)
    return merge(left, right)

# 数学公式：T(n) = c * n * log n
# 推导：递归深度 log n，每层合并操作 O(n)
```

##### O(n²) - 平方时间复杂度
**原理**：嵌套循环，常见于简单排序算法。

**示例**：冒泡排序
```python
def bubble_sort(arr):
    """O(n²) - 冒泡排序"""
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

# 数学公式：T(n) = c * n²
# 推导：外层循环 n 次，内层平均 n/2 次，总操作 ~ n²/2
```

##### O(2ⁿ) - 指数时间复杂度
**原理**：常见于暴力搜索、递归求斐波那契数列。

**示例**：递归斐波那契
```python
def fibonacci_recursive(n):
    """O(2ⁿ) - 递归斐波那契（低效版本）"""
    if n <= 1:
        return n
    return fibonacci_recursive(n-1) + fibonacci_recursive(n-2)

# 数学公式：T(n) ≈ c * 2ⁿ
# 推导：递归树有 ~2ⁿ 个节点
```

##### O(n!) - 阶乘时间复杂度
**原理**：全排列问题，如旅行商问题的暴力解法。

**示例**：生成全排列
```python
def generate_permutations(arr, start=0):
    """O(n!) - 生成全排列"""
    if start == len(arr) - 1:
        print(arr)
        return
    
    for i in range(start, len(arr)):
        arr[start], arr[i] = arr[i], arr[start]
        generate_permutations(arr, start + 1)
        arr[start], arr[i] = arr[i], arr[start]

# 数学公式：T(n) = c * n!
```

| 复杂度 | n=10 | n=100 | n=1000 | 增长趋势 | 典型算法 |
|--------|------|-------|--------|----------|----------|
| O(1) | 1 | 1 | 1 | 常数 | 数组访问 |
| O(log n) | ~3 | ~7 | ~10 | 非常慢 | 二分查找 |
| O(n) | 10 | 100 | 1000 | 线性 | 线性搜索 |

### 三、空间复杂度（Space Complexity）

#### 3.1 定义与分类
**定义**：算法运行过程中所需的内存空间随输入规模的增长趋势。

**分类**：
1. **原地算法**：O(1)，不依赖输入规模
2. **线性空间**：O(n)，如创建新数组
3. **递归栈空间**：O(h)，h为递归深度

#### 3.2 空间复杂度示例

##### O(1) - 常数空间
```python
def find_max(arr):
    """O(1)空间 - 只使用固定数量的变量"""
    max_val = arr[0]
    for num in arr:
        if num > max_val:
            max_val = num
    return max_val
# 只使用 max_val 一个额外变量，与数组大小无关
```

##### O(n) - 线性空间
```python
def copy_and_sort(arr):
    """O(n)空间 - 创建新数组"""
    sorted_arr = sorted(arr)  # 创建大小为n的新数组
    return sorted_arr
```

##### O(n²) - 平方空间
```python
def generate_matrix(n):
    """O(n²)空间 - 创建n×n矩阵"""
    matrix = [[0] * n for _ in range(n)]  # n²个元素
    return matrix
```

### 四、复杂度分析实战练习

#### 练习1：分析以下代码的时间复杂度
```python
def example1(n):
    total = 0
    for i in range(n):
        for j in range(n):
            total += i * j
    return total
```
**答案**：O(n²) - 双重嵌套循环

#### 练习2：分析以下代码的时间复杂度
```python
def example2(n):
    result = []
    for i in range(n):
        result.append(i)
        for j in range(10):
            result.append(j)
    return result
```
**答案**：O(n) - 内层循环固定10次，为常数项

#### 练习3：分析以下代码的时间和空间复杂度
```python
def example3(n):
    if n <= 0:
        return []
    
    matrix = [[0] * n for _ in range(n)]
    
    for i in range(n):
        for j in range(n):
            matrix[i][j] = i + j
    
    return matrix
```
**答案**：时间复杂度O(n²)，空间复杂度O(n²)

#### 练习4：递归复杂度分析
```python
def example4(n):
    if n <= 1:
        return 1
    return example4(n-1) + example4(n-2)
```
**答案**：时间复杂度O(2ⁿ)，空间复杂度O(n)（递归深度）

### 五、大厂面试常见复杂度问题

#### 5.1 必考知识点
1. **数组与链表操作复杂度对比**
2. **各种排序算法的时间/空间复杂度**
3. **哈希表的平均与最坏情况复杂度**
4. **树结构操作复杂度**

#### 5.2 复杂度分析技巧
1. **关注循环嵌套层级**：每层循环乘以n
2. **递归分析**：使用递归树或主定理
3. **均摊分析**：某些操作可能代价高但平均代价低
4. **最坏/平均/最好情况**：明确面试官问的是哪种

#### 5.3 实战面试题
1. **题目**：实现一个函数，判断数组中是否存在重复元素，分析复杂度。
   
   **最优解**：使用哈希表，时间复杂度O(n)，空间复杂度O(n)

2. **题目**：两个有序数组合并，分析复杂度。
   
   **最优解**：双指针法，时间复杂度O(m+n)，空间复杂度O(1)或O(m+n)
