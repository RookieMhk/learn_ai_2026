### 第1题 [基础] 监督学习的基本流程

**题目**：在监督学习中，模型训练的正确顺序是？

```
A. 数据清洗 → 特征工程 → 划分数据集 → 训练模型 → 评估模型
B. 划分数据集 → 数据清洗 → 特征工程 → 训练模型 → 评估模型
C. 数据清洗 → 划分数据集 → 特征工程 → 评估模型 → 训练模型
D. 特征工程 → 数据清洗 → 划分数据集 → 训练模型 → 评估模型
```

**答案**：**A**

**解析**：正确的流程应该是：
1. **数据清洗** - 处理缺失值、异常值
2. **特征工程** - 特征选择、特征变换
3. **划分数据集** - 训练集/验证集/测试集
4. **训练模型** - 使用训练集拟合模型
5. **评估模型** - 使用测试集评估泛化能力

⚠️ 注意：特征工程应在划分数据集之前完成，避免数据泄露。

### 第2题 [基础] 训练集与测试集的划分比例

**题目**：对于一个中等规模的数据集（10万条），以下哪个划分比例最合理？

```
A. 50:50
B. 60:20:20（训练:验证:测试）
C. 90:10
D. 99:1
```

**答案**：**B**

**解析**：
- **训练集过大**：可能导致过拟合，测试集不足则评估不准确
- **训练集过小**：模型学习不充分，欠拟合
- **标准比例**：通常采用 **70:15:15** 或 **80:10:10**
- **大数据集**：可以适当减少测试集比例，如95:2.5:2.5

## 🧠 模块二：注意力机制原理（5-7题）

### 第5题 [基础] 自注意力的核心计算

**题目**：在自注意力机制中，注意力分数的计算公式是？

```
A. Attention = softmax(Q · K)
B. Attention = softmax(Q · K^T / √d_k) · V
C. Attention = softmax(V · K^T)
D. Attention = Q + K + V
```

**答案**：**B**

**解析**：
```
注意力计算流程：

    Q ──┐
        ├──▶ 计算 Q·K^T ──▶ 除以 √d_k ──▶ softmax ──▶ 权重
    K ──┘                                              │
                                                      ▼
                                                  加权求和
                                                      │
    V ───────────────────────────────────────────────┘

公式：Attention(Q,K,V) = softmax(QK^T / √d_k) · V
```

- **Q·K^T**：计算Query和Key的相似度
- **√d_k**：缩放因子，防止点积过大导致梯度消失
- **softmax**：归一化得到注意力权重
- **乘以V**：加权求和得到最终输出

### 第6题 [中等] Q/K/V矩阵的来源

**题目**：Query、Key、Value矩阵是通过什么方式得到的？

```
A. 随机初始化后固定不变
B. 通过线性变换从输入 embedding 生成
C. 直接使用原始输入
D. 通过卷积操作得到
```

**答案**：**B**

**解析**：
```
输入 X ──▶ 线性变换 ──▶ Q, K, V

    X (batch, seq_len, d_model)
         │
         ├──▶ W_q ──▶ Q (batch, seq_len, d_k)
         │
         ├──▶ W_k ──▶ K (batch, seq_len, d_k)
         │
         └──▶ W_v ──▶ V (batch, seq_len, d_v)

W_q, W_k, W_v 是可学习的参数矩阵
```

⚠️ 注意：这些权重矩阵在训练过程中会不断更新学习。

## ⚙️ 模块三：损失函数与优化算法（8-11题）

### 第8题 [基础] 交叉熵损失函数公式

**题目**：对于二分类问题，交叉熵损失的正确形式是？

```
A. L = -y·log(ŷ)
B. L = -(y·log(ŷ) + (1-y)·log(1-ŷ))
C. L = |y - ŷ|
D. L = (y - ŷ)²
```

**答案**：**B**

**解析**：
```
二分类交叉熵：
    L = -[y·log(ŷ) + (1-y)·log(1-ŷ)]

其中：
    y = 真实标签 (0或1)
    ŷ = 预测概率 (0~1)

当 y=1 时：L = -log(ŷ)     → 预测概率越小，损失越大
当 y=0 时：L = -log(1-ŷ)  → 预测概率越大，损失越大
```

⚠️ 注意：实际实现时需要添加数值稳定性处理（epsilon）。

```
A. Momentum + AdaGrad

```

**答案**：**B**

**解析**：
```
Adam = Adaptive + Momentum

├── 一阶矩估计（Momentum）
│   └── 梯度指数加权移动平均
│
└── 二阶矩估计（RMSProp）

更新公式：
    m_t = β₁·m_{t-1} + (1-β₁)·g_t        # 动量
    v_t = β₂·v_{t-1} + (1-β₂)·g_t²      # 自适应学习率
    θ_t = θ_{t-1} - α·m_t/(√v_t + ε)    # 参数更新
```

### 第11题 [困难] 梯度消失问题的原因

**题目**：深度网络中梯度消失的主要原因是什么？

```
A. 学习率设置过大
B. 激活函数在极端值处梯度接近零
C. 网络层数太少
D. 数据量不足
```

**答案**：**B**

**解析**：
```
梯度消失原因：

以 Sigmoid 为例：
    σ(x) = 1/(1+e^(-x))
    
    导数：σ'(x) = σ(x)(1-σ(x))
    
    最大值在 x=0 处：σ'(0) = 0.25
    
问题：
    反向传播时：
    ∂L/∂x₁ = σ'(x₁)·W₁·σ'(x₂)·W₂·...·σ'(x_n)
    
    每层乘以 ≤0.25 的梯度，
    深层网络的梯度趋近于零！

解决方案：
    ✓ ReLU激活函数（梯度为0或1）
    ✓ 残差连接（ResNet）
    ✓ 批量归一化
    ✓ LSTM/GRU门控机制
```

### 第13题 [中等] Precision与Recall的计算

**题目**：某模型在测试集上的结果为：TP=80, TN=15, FP=5, FN=0
则 Precision 和 Recall 分别是多少？

```
A. Precision=80%, Recall=94%
B. Precision=94%, Recall=100%
C. Precision=100%, Recall=80%
D. Precision=94%, Recall=80%
```

**答案**：**B**

**解析**：
```
Precision（精确率）：
    = TP / (TP + FP)
    = 80 / (80 + 5)
    = 80/85 ≈ 94.1%

Recall（召回率）：
    = TP / (TP + FN)
    = 80 / (80 + 0)
    = 80/80 = 100%

解释：
    Precision = "预测为正的样本中，有多少是真正的正例"
    Recall = "所有正例中，有多少被正确预测出来"
```

### 第14题 [中等] F1-Score的调和平均特性

**题目**：F1-Score为什么使用调和平均而不是算术平均？

```
A. 调和平均计算更快
B. 调和平均对极低值更敏感
C. 调和平均对极高值更敏感
D. 调和平均更易于理解
```

**答案**：**B**

**解析**：
```
算术平均 vs 调和平均：

    算术平均：F1_arith = (P + R) / 2
    调和平均：F1 = 2·P·R / (P + R)

示例（极端情况）：
    P=1.0, R=0.0
    
    算术平均：F1_arith = (1.0 + 0.0) / 2 = 0.5  ❌ 不公平！
    调和平均：F1 = 2×1.0×0.0 / (1.0+0.0) = 0.0  ✅ 正确反映

调和平均的特性：
    ✓ 对极低值惩罚更重
    ✓ 只有P和R都高时，F1才会高
    ✓ 避免"一高遮百丑"的问题
```

### 第15题 [困难] ROC-AUC的含义

**题目**：AUC=0.7 表示什么意思？

```
A. 70%的样本被正确分类
B. 随机挑选一个正例和一个负例，70%的概率正例排在负例前面
C. 模型的准确率是70%
D. 召回率是70%
```

**答案**：**B**

**解析**：
```
ROC-AUC 的定义：

    AUC = P(正例得分 > 负例得分)

含义：
    随机抽取一个正样本和一个负样本，
    模型给正样本打分高于负样本的概率。

AUC取值范围：
    AUC = 0.5  → 随机猜测（红虚线）
    AUC = 0.7  → 70%概率区分正负样本
    AUC = 1.0  → 完美分类

    TPR
      ↑
    1 ┤╲
      │  ╲
      │   ╲    AUC = 0.7
      │    ╲
      │     ╲
      │──────╲────────
    0 └────────────────→ FPR
      0    0.5         1
```

### 第16题 [基础] 标准化 vs 归一化

**题目**：以下哪个选项正确描述了标准化(Z-score)和归一化(Min-Max)的区别？

```
A. 两者没有区别
B. 标准化将数据映射到[0,1]，归一化使数据服从正态分布
C. 标准化使数据服从正态分布，归一化将数据映射到[0,1]
D. 标准化适用于树模型，归一化适用于神经网络
```

**答案**：**C**

**解析**：
```
归一化（Min-Max Scaling）：
    x' = (x - min) / (max - min)
    → 映射到 [0, 1] 范围
    → 对异常值敏感

标准化（Z-score）：
    x' = (x - μ) / σ
    → 服从均值为0、标准差为1的正态分布
    → 对异常值更鲁棒

选择建议：
    归一化：数据分布均匀，无明显异常值
    标准化：数据近似正态分布，存在异常值
```

### 第17题 [中等] One-Hot编码的目的

**题目**：为什么不能直接对类别特征使用数字编码（如 1,2,3）？

```
A. 数字编码太占用内存
B. 模型会错误学习类别之间的数值关系
C. 数字编码无法处理缺失值
D. One-Hot编码更快
```

**答案**：**B**

**解析**：
```
问题：数字编码隐含了大小关系

示例：城市特征 [北京=1, 上海=2, 广州=3]

    模型可能学到：上海 = 北京 + 广州
    
    但实际上三个城市是平等的分类！

解决方案：One-Hot编码

    北京 ──▶ [1, 0, 0]
    上海 ──▶ [0, 1, 0]
    广州 ──▶ [0, 0, 1]

⚠️ 注意：
    - 类别数量多时使用独热编码会导致维度爆炸
    - 可考虑目标编码、嵌入等替代方案
```

### 第19题 [中等] 解决过拟合的策略

**题目**：以下哪项措施不能有效解决过拟合问题？

```

```

**答案**：**D**

**解析**：
```

✅ 简化模型：减少层数/参数
✅ Dropout：随机丢弃神经元
✅ Early Stopping：监控验证集性能
✅ 交叉验证：更稳定评估模型

❌ 减少验证集：降低评估可靠性，无法解决过拟合
```

### 第20题 [困难] 偏差-方差分解

**题目**：在高偏差状态下，增加训练数据会有什么效果？

```
A. 显著提升模型性能
B. 对模型性能提升有限
C. 导致过拟合
D. 无法确定
```

**答案**：**B**

**解析**：
```
偏差-方差分解：

    总误差 = 偏差² + 方差 + 噪声

┌─────────────────────────────────────────┐
│                                         │
│           偏差-方差权衡图                │
│                                         │
│     误差                                
│       ↑                                 
│       │    /←高方差→\                  │
│       │   /          \                 │
│       │  /            \                │
│       │ /              \               │
│       │/                \              │
│       └────────────────────→ 复杂度    │
│                                         │
│  低复杂度： 高偏差 + 低方差（欠拟合）      │
│  高复杂度： 低偏差 + 高方差（过拟合）      │
│                                         │
└─────────────────────────────────────────┘

高偏差（欠拟合）时：
    → 增加数据效果有限
    → 应该增加模型复杂度或改进模型结构

高方差（过拟合）时：
    → 增加数据会有帮助
    → 应该正则化或简化模型
```
