# AI学习每周循环执行模式说明

## 一、模式概述

本执行模式专为「3个月AI学习计划」设计，基于用户确认的内容形态（深度文档+代码Notebook+知识测验+可视化图表+每日动态简报）和验收机制，构建了「计划-执行-复盘」的闭环学习系统。

**核心要素**：
- **时间周期**：每周7天，适用于Week 1-12
- **每日投入**：约1.5-2小时（每周8-12小时）
- **内容形态**：5种交付形态组合
- **动态更新**：每日AI技术动态跟踪 + 每周计划调整

## 二、每周执行模板（Day 1-7）

### 2.1 每日固定任务：输出《今日执行卡片》
**执行时间**：每日学习结束后
**格式**：Markdown文档
**内容要求**：
```
### 今日执行卡片 - [日期]
#### 3个核心收获
1. [收获一：学到的具体知识点]
2. [收获二：掌握的技能或方法]
3. [收获三：对AI技术的理解深化]

#### 1个疑问/困难
- [学习中遇到的难点或疑问]

#### 1个感悟/调整
- [学习感悟或计划调整思路]

#### 今日AI动态摘要
- [简要记录今日关注的AI技术动态]
```

### 2.2 Day 1, 3, 5：深度输入日
**时间分配**：约2小时
**核心目标**：理论学习 + 前沿动态跟踪

**具体任务流程**：
1. **AI动态搜索**（10分钟）
   - 搜索最新AI技术动态（大模型进展、算法突破、行业应用）
   - 筛选1-3个与本周学习主题相关的重点动态

2. **深度文档学习**（60分钟）
   - 阅读《[本周主题]深度文档》
   - 重点理解：数学原理、算法思想、应用场景
   - 结合代码示例理解抽象概念

3. **AI动态分析**（20分钟）
   - 阅读深度文档中的「今日AI动态」栏目
   - 思考：该技术动态与本周学习内容的关联
   - 记录对职业发展的启发

4. **知识点整理**（30分钟）
   - 整理学习笔记，绘制概念关系图
   - 标记重点和难点，为实战日做准备

**产出要求**：
- 完成深度文档学习与理解
- 记录「今日AI动态」分析思考
- 整理个人学习笔记

### 2.3 Day 2, 4, 6：实战演练日
**时间分配**：约2小时
**核心目标**：代码实践 + 项目推进

**具体任务流程**：
1. **代码练习**（45分钟）
   - 运行代码Notebook中的示例
   - 修改参数、调试代码、观察结果变化
   - 完成Notebook中的挑战任务

2. **可视化分析**（30分钟）
   - 生成数据可视化图表
   - 分析图表反映的数学规律
   - 截图或保存重要可视化结果

3. **项目任务**（45分钟）
   - 推进本周阶段性项目
   - 实现特定功能模块
   - 测试代码正确性，优化性能

**产出要求**：
- 完成所有代码练习任务
- 生成可视化图表文件
- 推进项目进展，提交代码更新

### 2.4 Day 7：复盘评估日
**时间分配**：约2.5小时
**核心目标**：周度复盘 + 知识测验 + 计划调整

**具体任务流程**：
1. **学习成果整理**（40分钟）
   - 汇总6份《今日执行卡片》
   - 提炼本周核心知识点体系
   - 生成《周进展脑图》（思维导图格式）

2. **知识测验**（40分钟）
   - 完成本周5-10道测验题
   - 检查答案解析，分析错误原因
   - 标记薄弱知识点

3. **项目评审**（40分钟）
   - 评审本周项目进展
   - 识别技术难点和优化空间
   - 制定下周项目推进计划

4. **动态趋势分析**（30分钟）
   - 回顾本周AI技术动态
   - 分析技术发展趋势
   - 识别对学习计划的影响

5. **下周计划调整**（20分钟）
   - 基于本周进展调整下周学习重点
   - 根据技术动态更新实践项目方向
   - 制定详细的Week [N+1]任务清单

**产出要求**：
- 《周进展脑图》文档
- 知识测验成绩与错题分析
- 项目评审报告
- Week [N+1]任务清单

## 三、动态更新机制

### 3.1 每日AI动态简报生成流程
**触发条件**：每日深度输入日自动执行
**搜索策略**：
1. **关键词组合**：[本周主题] + 最新进展/2026年/突破
2. **权威源优先**：学术论文、官方博客、行业报告
3. **时效性过滤**：优先最近7天的内容

**内容生成标准**：
1. **精选1-3个动态**：避免信息过载
2. **结构清晰**：技术概述 + 核心突破 + 行业影响
3. **学习关联**：明确与本周学习主题的连接点
4. **实践启发**：提供可能的代码实现或项目思路

**集成方式**：作为深度文档的第X章「今日AI动态」

### 3.2 每周计划调整决策逻辑
**调整触发条件**：周日复盘完成后
**决策输入**：
1. **学习进展数据**：知识测验成绩、代码完成度、项目里程碑
2. **技术动态趋势**：本周关注的AI技术发展方向
3. **用户反馈**：执行卡片中的疑问和难点
4. **大厂需求变化**：最新招聘要求和技术栈更新

**调整决策矩阵**：
| 评估维度 | 进展良好 | 进展一般 | 进展滞后 |
|---------|---------|---------|---------|
| **保持原计划** | 按计划推进 | 微调重点 | 强化基础 |
| **加速推进** | 增加挑战 | 调整节奏 | 重点突破 |
| **方向调整** | 深化前沿 | 优化方法 | 简化目标 |

**调整输出**：
1. **下周主题微调**：基于技术动态增加/减少某些知识点
2. **难度系数调整**：根据掌握程度调整练习难度
3. **项目优先级**：基于行业趋势调整项目技术栈
4. **资源推荐更新**：补充最新优质学习资源

### 3.3 LLM技术融入机制
**渐进式融入策略**：
- **Week 1-3**：LLM基础概念（Transformer架构、注意力机制）
- **Week 4-6**：LLM微调与实践（LoRA、RLHF）
- **Week 7-9**：LLM工程化（部署、优化、Agent架构）
- **Week 10-12**：LLM前沿应用（RAG、MCP、多模态）

**每周LLM元素**：
- 至少一个任务明确涉及LLM相关知识
- 代码练习包含LLM相关API调用或实现
- 项目任务逐步引入LLM应用场景

## 四、验收标准体系

### 4.1 每日验收标准
**《今日执行卡片》质量要求**：
- ✅ 3个收获具体、明确，与学习内容强相关
- ✅ 1个疑问真实反映学习难点
- ✅ 1个感悟有深度，体现学习反思
- ✅ AI动态摘要准确、简洁

**理论学习验收**：
- ✅ 理解深度文档中80%以上核心概念
- ✅ 能复述关键数学原理和算法思想
- ✅ 建立知识点之间的逻辑联系

**代码实践验收**：
- ✅ 完成所有代码练习任务
- ✅ 代码运行正确，无语法错误
- ✅ 理解代码背后的数学原理

### 4.2 每周验收标准
**知识测验**：
- ✅ 完成率100%
- ✅ 正确率≥80%
- ✅ 错题分析深入，有改进计划

**项目进展**：
- ✅ 完成本周项目里程碑
- ✅ 代码质量符合工程标准
- ✅ 技术文档完整、清晰

**学习体系**：
- ✅ 《周进展脑图》结构清晰，逻辑完整
- ✅ 建立了本周知识点的体系化理解
- ✅ 能关联不同知识点，形成知识网络

### 4.3 阶段性验收标准
**第一阶段结束**（第3周）：
- ✅ 掌握线性代数、概率统计核心概念
- ✅ 能手动实现线性回归、逻辑回归
- ✅ 熟练使用Scikit-learn完成完整机器学习流程

**第二阶段结束**（第6周）：
- ✅ 理解神经网络基本原理
- ✅ 能使用PyTorch实现CNN图像分类
- ✅ 掌握Transformer架构核心组件

**第三阶段结束**（第9周）：
- ✅ 掌握模型部署和优化技术
- ✅ 能构建Docker容器化AI服务
- ✅ 理解MCP、Agent、RAG等前沿技术原理

**第四阶段结束**（第12周）：
- ✅ 完成2-3个高质量AI项目
- ✅ 建立完整的GitHub作品集
- ✅ 具备大厂AI岗位面试所需技术能力

## 五、文件目录与命名规范

### 5.1 周度执行目录结构
```
outputs/
├── 阶段一/                    # 第一阶段（Week 1-3）
│   ├── Week1/                # 第一周
│   │   ├── Day1/             # 每日独立目录
│   │   │   ├── 深度文档/
│   │   │   ├── 代码练习/
│   │   │   ├── 学习笔记/
│   │   │   └── 今日执行卡片.md
│   │   ├── Day2/
│   │   │   └── ...
│   │   ├── Day7/
│   │   │   ├── 周进展脑图.md
│   │   │   ├── 知识测验.md
│   │   │   ├── 项目评审.md
│   │   │   └── Week2任务清单.md
│   │   └── 周度汇总/

│   │       └── Week1项目代码/
│   ├── Week2/
│   └── Week3/
├── 阶段二/                    # 第二阶段（Week 4-6）
├── 阶段三/                    # 第三阶段（Week 7-9）
└── 阶段四/                    # 第四阶段（Week 10-12）
```

### 5.2 文件命名规范
**深度文档**：`[主题]深度文档.md`（如：`线性代数核心概念深度文档.md`）
**代码练习**：`[主题]实战练习.py` 或 `[主题]代码挑战.ipynb`
**知识测验**：`Week[N]知识测验.md`
**今日执行卡片**：`[YYYY-MM-DD]今日执行卡片.md`
**周进展脑图**：`Week[N]周进展脑图.md`
**任务清单**：`Week[N]任务清单.md`

## 六、执行保障与支持

**每日监控**：执行卡片提交时间与质量
**每周监控**：测验成绩趋势、项目里程碑达成
**阶段监控**：技能掌握度评估、技术能力成长曲线

### 6.2 难点支持策略
**即时答疑**：执行卡片中的疑问24小时内响应
**专项辅导**：针对薄弱知识点提供补充材料
**代码调试**：协助解决编程中的技术难题

### 6.3 动态优化机制
**每周评估**：基于数据和反馈优化执行流程
**每月评审**：全面评估学习效果，调整整体策略
**持续改进**：根据技术发展更新学习内容和方法
