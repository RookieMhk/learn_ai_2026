## 实验一：矩阵乘法性能对比

### 实验设计
对比手动三重循环实现与NumPy内置`np.matmul`在不同规模矩阵下的性能差异。

### 实验步骤
1. **生成测试矩阵**：创建不同规模的随机矩阵（10×10, 50×50, 100×100, 200×200）
2. **手动实现计时**：使用三重循环计算矩阵乘法，记录运行时间
3. **NumPy实现计时**：使用`np.matmul`计算矩阵乘法，记录运行时间
4. **结果验证**：计算两种实现结果的差异，确保算法正确性
5. **性能分析**：计算加速比（手动时间/NumPy时间）

### 关键代码片段
```python
import numpy as np
import time

def manual_matrix_multiply(A, B):
    m, n = A.shape
    n2, p = B.shape
    assert n == n2
    C = np.zeros((m, p))
    for i in range(m):
        for j in range(p):
            sum_val = 0
            for k in range(n):
                sum_val += A[i, k] * B[k, j]
            C[i, j] = sum_val
    return C

# 性能测试
sizes = [10, 50, 100, 200]
for size in sizes:
    A = np.random.randn(size, size)
    B = np.random.randn(size, size)
    
    # 手动实现
    start = time.time()
    C_manual = manual_matrix_multiply(A, B)
    manual_time = time.time() - start
    
    # NumPy实现
    start = time.time()
    C_numpy = np.matmul(A, B)
    numpy_time = time.time() - start
    
    speedup = manual_time / numpy_time
    print(f"Size {size}×{size}: Manual {manual_time:.4f}s, NumPy {numpy_time:.6f}s, Speedup {speedup:.1f}x")
```

### 预期结果分析
| 矩阵大小 | 手动时间 | NumPy时间 | 加速比 | 理论复杂度 |
|---------|---------|-----------|--------|-----------|
| 10×10   | ~0.001s | ~0.00001s | ~100x  | O(10³)=1000 |
| 50×50   | ~0.05s  | ~0.0001s  | ~500x  | O(50³)=125000 |
| 100×100 | ~0.4s   | ~0.0005s  | ~800x  | O(100³)=1e6 |
| 200×200 | ~3.2s   | ~0.003s   | ~1000x | O(200³)=8e6 |

**关键洞察**：
- 矩阵乘法时间复杂度为O(n³)，规模增大时计算量急剧增加
- NumPy使用BLAS/LAPACK库、SIMD指令、多线程等优化，性能远超手动实现
- 向量化运算避免了Python解释器开销，直接在C/Fortran层执行

## 实验二：特征值计算对比

### 实验设计
对比幂迭代法（手动实现）与NumPy `np.linalg.eig`的性能和精度差异。

### 实验步骤
1. **生成对称测试矩阵**：创建8×8对称矩阵
2. **幂迭代法实现**：实现Rayleigh商迭代，记录收敛过程
3. **NumPy特征值计算**：使用`np.linalg.eig`计算所有特征值
4. **结果对比**：比较主特征值的精度差异
5. **收敛性分析**：绘制迭代过程中的特征值估计变化

### 关键代码片段
```python
def power_iteration(A, num_iterations=100):
    n = A.shape[0]
    b_k = np.random.rand(n)
    b_k = b_k / np.linalg.norm(b_k)
    
    for i in range(num_iterations):
        b_k1 = np.dot(A, b_k)
        eigenvalue = np.dot(b_k, b_k1)
        b_k = b_k1 / np.linalg.norm(b_k1)
        
        # 可选：检查收敛条件
        if i > 0 and abs(eigenvalue - prev_eigenvalue) < 1e-10:
            break
        prev_eigenvalue = eigenvalue
    
    return eigenvalue, b_k

# 对比分析
max_eigenvalue_manual, max_eigenvector_manual = power_iteration(M, 50)
eigenvalues_np, eigenvectors_np = np.linalg.eig(M)
max_eigenvalue_np = np.max(np.abs(eigenvalues_np))

print(f"手动计算最大特征值: {max_eigenvalue_manual:.6f}")
print(f"NumPy计算最大特征值: {max_eigenvalue_np:.6f}")
print(f"误差: {abs(max_eigenvalue_manual - max_eigenvalue_np):.6f}")
```

### 预期结果分析
- **幂迭代法**：仅计算绝对值最大的特征值，迭代50次后误差应小于1e-6
- **收敛速度**：与特征值间隙（λ₁/λ₂）有关，间隙越大收敛越快
- **应用场景**：适合大型稀疏矩阵的主特征值计算
- **NumPy优势**：同时计算所有特征值，精度更高（使用QR算法等数值稳定方法）

## 实验三：SVD分解对比

### 实验设计
对比简化版SVD（通过特征值分解）与NumPy `np.linalg.svd`的性能和精度差异。

### 实验步骤
1. **生成测试矩阵**：创建4×3矩形矩阵
2. **简化版SVD实现**：通过计算AᵀA和AAᵀ的特征值分解获取奇异值
3. **NumPy SVD计算**：使用`np.linalg.svd`进行完整分解
4. **重构验证**：使用前k个奇异值重构矩阵，计算重构误差
5. **奇异值对比**：比较两种方法计算的奇异值差异

### 关键代码片段
```python
def simple_svd(A, k=2):
    # 计算 AᵀA 和 AAᵀ
    ATA = np.dot(A.T, A)
    AAT = np.dot(A, A.T)
    
    # 特征值分解
    eigvals_ATA, eigvecs_ATA = np.linalg.eig(ATA)
    eigvals_AAT, eigvecs_AAT = np.linalg.eig(AAT)
    
    # 取前k个
    idx_ATA = np.argsort(eigvals_ATA)[::-1][:k]
    idx_AAT = np.argsort(eigvals_AAT)[::-1][:k]
    
    singular_values = np.sqrt(np.abs(eigvals_ATA[idx_ATA]))
    U = eigvecs_AAT[:, idx_AAT]
    Sigma = np.diag(singular_values)
    Vt = eigvecs_ATA[:, idx_ATA].T
    
    return U, Sigma, Vt

# 对比分析
U_simple, Sigma_simple, Vt_simple = simple_svd(X, k=2)
U_np, Sigma_np, Vt_np = np.linalg.svd(X, full_matrices=False)

print(f"简化版SVD奇异值: {Sigma_simple.diagonal()}")
print(f"NumPy SVD奇异值: {Sigma_np[:2]}")
```

### 预期结果分析
- **简化版SVD局限性**：仅计算部分奇异值，精度受特征值分解影响
- **数值稳定性**：NumPy使用分治算法，对病态矩阵更稳定
- **计算复杂度**：简化版O(n³)，NumPy使用更优算法（如分治算法O(n²)）
- **实际应用**：大型矩阵使用随机SVD、截断SVD等近似方法
