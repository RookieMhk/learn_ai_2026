## 一、语言模型的基本定义

### 1.1 什么是语言模型？

**语言模型（Language Model, LM）** 是一个对语言进行**概率建模**的系统，其核心任务是：**给定一个词序列，预测下一个词出现的概率**。

数学表达：
```
P(w₁, w₂, ..., wₙ) = P(w₁) × P(w₂|w₁) × ... × P(wₙ|w₁, ..., wₙ₋₁)
```

即：**联合概率 = 条件概率的连乘**

### 1.2 N-gram 模型：最早的语言模型

**核心思想**：用前 N-1 个词预测第 N 个词

| 模型 | 公式 | 特点 |
|------|------|------|
| Unigram | P(w) | 假设词之间独立 |
| Bigram | P(wₙ\|wₙ₋₁) | 只看前一个词 |
| Trigram | P(wₙ\|wₙ₋₂,wₙ₋₁) | 看前两个词 |

**问题**：数据稀疏严重，无法处理长距离依赖

### 1.3 神经语言模型

2003年，Bengio等人首次提出**神经网络语言模型**：

```
输入词 → 嵌入层 → 隐藏层 → 输出层 → 下一个词的概率
```

这是深度学习用于NLP的开端。

## 二、大语言模型（LLM）的定义

### 2.1 什么算"大"？

| 阶段 | 参数量 | 代表模型 | 时间 |
|------|--------|---------|------|
| 小型 | < 1B | ELMo | 2018 |
| 中型 | 1B - 10B | BERT-large | 2019 |
| 大型 | 10B - 100B | GPT-3 (175B) | 2020 |
| 超大 | > 100B | GPT-4, Claude, LLaMA | 2022-2026 |

**2026年最新**：GPT-6拥有5-6万亿参数，上下文窗口达200万Token

### 2.2 LLM的核心特征

1. **涌现能力（Emergent Abilities）**
   - 小模型没有，大模型突然具备的能力
   - 如：思维链推理、代码生成、多语言能力

2. **泛化能力**
   - 能在训练数据之外的任务上表现良好
   - 称为"零样本"或"少样本"学习

3. **通用性**
   - 一个模型解决多种任务
   - 不同于针对特定任务训练的专用模型

## 三、LLM的工作原理

### 3.1 预训练（Pre-training）

**目标**：在海量文本上学习"预测下一个词"

```
训练数据：互联网上的万亿级文本
训练目标：最大化下一个词预测的准确率
```

**关键数据来源**：
- 网页文本（Common Crawl）
- 书籍（BooksCorpus, GPT-3 books）
- 代码（GitHub）
- 学术论文（ArXiv）
- 对话数据

### 3.2 词元化（Tokenization）

**Token**：模型处理的基本单位

```
文本："机器学习真有趣"
       ↓ Tokenize
Token: ["机器", "学习", "真", "有趣"]  # 4个token
       ↓ 映射为ID
IDs:   [1024, 2048, 512, 3072]
```

**常见分词器**：
- BPE (Byte Pair Encoding) - GPT系列
- WordPiece - BERT
- SentencePiece - 通用的子词分词器

### 3.3 Transformer架构

LLM的核心是**Transformer解码器**：

```
输入Token → 词嵌入 → 位置编码 → [解码器层 × N] → 下一个Token概率
```

**关键组件**：
- 自注意力机制：理解上下文关系
- 前馈网络：非线性变换
- 残差连接 + 层归一化：稳定训练

## 四、LLM的关键技术

### 4.1 缩放定律（Scaling Laws）

**发现**：模型性能随参数量、数据量、计算量的增加而提升

```
Kaplan et al. (2020) 发现的规律：
- L损失 ∝ N^(-0.076)  （N = 参数数量）
- L损失 ∝ D^(-0.095)  （D = 数据量）
- L损失 ∝ C^(-0.054)  （C = 计算量）
```

**启示**：增大模型、数据、算力都能提升性能

### 4.2 指令微调（Instruction Tuning）

**目的**：让LLM能够理解和执行人类指令

```
格式：
[系统提示] → 设定角色和行为
[用户指令] → 具体的任务描述
[模型输出] → 期望的回答
```

**示例**：
```
系统：你是一个有帮助的助手
用户：把"今天天气真好"翻译成英文
模型：The weather is really nice today.
```

### 4.3 对齐技术（Alignment）

**目标**：让LLM的输出符合人类价值观和期望

**核心技术**：

| 技术 | 描述 | 代表 |
|------|------|------|
| RLHF | 基于人类反馈的强化学习 | ChatGPT |
| RLAIF | 基于AI反馈的强化学习 | Claude |
| DPO | 直接偏好优化 | 开源模型常用 |
| GRPO | 组相对策略优化 | DeepSeek |

### 4.4 上下文学习（In-Context Learning）

**定义**：LLM能够从提示（Prompt）中的例子学习新任务

```
示例：
输入：
"狗 → 动物
猫 →
"

输出："动物"
```

**不需要更新权重**，直接通过提示指定任务。

## 六、LLM的局限性

### 6.1 已知问题

| 问题 | 描述 | 缓解方法 |
|------|------|----------|
| **幻觉** | 生成看似合理但错误的内容 | RAG, 检索增强 |
| **知识截止** | 无法获取最新信息 | 联网搜索 |
| **推理成本** | 大模型推理耗时耗钱 | 量化, 蒸馏 |
| **长度限制** | 无法处理超长上下文 | 长上下文模型 |
| **偏见** | 反映训练数据中的偏见 | 对齐微调 |

### 6.2 幻觉问题的深层理解

**为什么LLM会"一本正经地胡说八道"？**

1. **本质原因**：LLM是"概率机器"，不是"知识库"
   - 它生成的是**概率上最可能的文本**，不是**事实**
   
2. **训练数据问题**：
   - 训练语料中可能有错误信息
   - 某些领域数据稀缺

3. **上下文压缩**：
   - 无法记住所有细节
   - 可能混淆相似事件
