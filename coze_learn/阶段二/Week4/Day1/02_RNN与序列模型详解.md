## 1. RNN基础原理与梯度问题

### 1.1 RNN的核心思想

循环神经网络的核心创新在于**隐藏状态的时间传播机制**，赋予神经网络"记忆"能力，使其能够处理任意长度的序列数据。

1990年，Jeffrey Elman提出了简单循环网络(SRN)，引入了将前一时间步的隐藏状态传回当前时间步的设计。

### 1.2 数学公式

RNN的前向传播：

```
h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b_h)
y_t = W_hy · h_t + b_y
```

其中：
- `h_t` = 时间步t的隐藏状态（记忆）
- `x_t` = 时间步t的输入
- `y_t` = 时间步t的输出
- `W_hh, W_xh, W_hy` = 权重矩阵（在所有时间步共享）

**关键优势**：权重共享——无论序列多长，模型参数量保持不变。

### 1.3 梯度消失问题

**理论 vs 现实**：理论上RNN可以捕捉任意长距离依赖，但1994年Bengio等人的研究揭示了残酷现实：标准RNN难以学习超过10-20步的长期依赖。

**数学根源**：时间反向传播(BPTT)中，梯度需要穿过多个时间步：

```
∂L/∂h_0 = ∂L/∂h_T · ∂h_T/∂h_{T-1} · ... · ∂h_1/∂h_0
```

每个 `∂h_t/∂h_{t-1}` 项涉及权重矩阵 W_hh 的重复乘法：
- 若 W_hh 最大特征值 < 1 → 梯度指数衰减（梯度消失）
- 若 W_hh 最大特征值 > 1 → 梯度指数增长（梯度爆炸）

**直观理解**：要让RNN记住第1个词来预测第100个词，梯度必须穿越99个时间步。每次乘以小于1的值，99次后梯度接近于零——网络"学不会"这种长距离依赖。

## 2. LSTM门控机制详解

### 2.1 核心创新

1997年，Sepp Hochreiter和Jurgen Schmidhuber提出长短期记忆网络(LSTM)，通过精巧的门控机制优雅地解决了梯度消失问题。

**核心创新**：引入"记忆高速公路"（细胞状态），允许信息跨多个时间步无损流动。

### 2.2 LSTM组件详解

| 组件 | 公式 | 功能 |
|------|------|------|
| **遗忘门** f_t | σ(W_f · [h_{t-1}, x_t] + b_f) | 决定"忘记什么"——从细胞状态中移除过时信息 |
| **输入门** i_t | σ(W_i · [h_{t-1}, x_t] + b_i) | 决定"记住什么"——写入新信息到细胞状态 |
| **候选记忆** C̃_t | tanh(W_C · [h_{t-1}, x_t] + b_C) | 生成新的候选记忆内容 |
| **细胞状态** C_t | f_t ⊙ C_{t-1} + i_t ⊙ C̃_t | 更新记忆：遗忘旧信息 + 添加新信息 |
| **输出门** o_t | σ(W_o · [h_{t-1}, x_t] + b_o) | 决定"输出什么"——从细胞状态中读取 |
| **隐藏状态** h_t | o_t ⊙ tanh(C_t) | 当前时间步的输出 |

### 2.3 为什么LSTM能解决梯度消失？

关键在于细胞状态的更新公式 `C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t`：

- 这是**加性结构**而非乘性结构
- 梯度可以沿着细胞状态**直接传播**，无需持续乘法
- 当遗忘门接近1时，梯度几乎无损耗地传向遥远的过去

### 2.4 门控机制的本质

三个门控本质上是在学习**信息流控制策略**：
- 遗忘门 ≈ "什么信息不再重要？"
- 输入门 ≈ "什么新信息值得记住？"
- 输出门 ≈ "基于当前状态，应该输出什么？"

## 3. GRU简化设计

### 3.1 设计理念

2014年，Cho等人提出门控循环单元(GRU)，可视为LSTM的简化版本。

**核心简化**：
1. 将遗忘门和输入门合并为单一的**更新门**
2. 消除独立的细胞状态，只保留隐藏状态
3. 参数量减少约25%

### 3.2 GRU公式

```
# 更新门：保留多少旧记忆？
z_t = σ(W_z · [h_{t-1}, x_t])

# 重置门：忘记多少过去？
r_t = σ(W_r · [h_{t-1}, x_t])

# 候选隐藏状态
h̃_t = tanh(W · [r_t ⊙ h_{t-1}, x_t])

# 最终隐藏状态：融合旧记忆和新信息
h_t = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t
```

### 3.3 LSTM vs GRU 对比

| 特性 | LSTM | GRU |
|------|------|-----|
| 门数量 | 3个（遗忘、输入、输出） | 2个（更新、重置） |
| 记忆机制 | 独立的细胞状态 | 直接在隐藏状态中 |
| 参数量 | 较多 | 较少（约75%） |
| 计算效率 | 较低 | 较高 |
| 表达能力 | 更强（更灵活） | 稍弱（但足够大多数任务） |

**实践建议**：
- 资源受限或需快速推理 → GRU
- 任务复杂或需精细记忆控制 → LSTM

## 4. 序列到序列(Seq2Seq)架构

### 4.1 基本结构

Seq2Seq模型是现代NLP的基石，源于2014年Sutskever等人的工作。核心思想：

```
输入序列 → [Encoder] → 上下文向量 → [Decoder] → 输出序列
```

### 4.2 Encoder-Decoder范式

**Encoder**：
- 将变长输入序列编码为固定维度的上下文向量
- 通常使用LSTM/GRU处理输入，逐步压缩信息

**Decoder**：
- 基于上下文向量逐步生成输出序列
- 自回归生成：每个时间步的输入是上一步的输出

### 4.3 注意力机制

标准Seq2Seq的瓶颈：所有信息压缩到单一上下文向量，造成信息瓶颈。

**注意力机制**（Bahdanau, 2014）解决了这一问题：

```
注意力得分 = Score(h_t, h_s)  # Decoder状态 vs Encoder所有状态
注意力权重 = Softmax(注意力得分)
上下文向量 = Σ(注意力权重 × Encoder隐藏状态)
```

**本质**：让Decoder在生成每个词时，能够"关注"输入序列的相关部分。

### 4.4 从Seq2Seq到Transformer

Seq2Seq + 注意力 = 原始Transformer的核心

- 编码器：双向自注意力
- 解码器：掩码自注意力 + 交叉注意力
- 位置编码：替代RNN的时间建模

## 5. 前沿进展：RNN的复兴

### 5.1 2025年Bengio团队：minLSTM与minGRU

**核心创新**：去除隐藏状态依赖，支持并行训练

**方法**：
1. 将门的计算从 `[h_{t-1}, x_t]` 简化为仅依赖 `x_t`
2. 去掉tanh的范围限制
3. 支持并行扫描算法训练

**结果**：
- 训练速度提升2-8倍
- 参数量显著减少
- 性能与Mamba等先进模型相当

### 5.2 AttnRNN：注意力+循环的融合

**设计原则**：
- 注意力竞争机制：历史信息与新输入直接竞争
- 残差主导：确保历史信息占主导地位
- 单门控设计：减少参数

**性能**：在长序列任务中全面超越LSTM/GRU，同时参数量相当或更少。

### 5.3 混合架构趋势

- **AttentionGRU**：结合Transformer的注意力与GRU的效率
- **LSTM-GRU混合**：实验表明混合架构在时序预测中表现最优
- **Titans架构**：Google提出的神经长期记忆模块，测试时扩展至200万上下文

## 关键概念速查

| 概念 | 核心要点 |
|------|----------|
| **RNN** | 权重共享，但面临梯度消失 |
| **LSTM** | 三门控+细胞状态，解决长期依赖 |
| **GRU** | 双门控，更轻量，性能接近LSTM |
| **Seq2Seq** | Encoder-Decoder架构 |
| **注意力** | 解决信息瓶颈，Transformer基础 |
| **梯度问题** | RNN无法学习长依赖，LSTM/GRU的门控解决了这一问题 |
