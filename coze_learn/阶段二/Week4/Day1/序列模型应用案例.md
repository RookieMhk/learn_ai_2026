## 1. 时间序列预测

### 1.1 应用场景

时间序列预测是最经典的RNN应用场景，包括：

| 领域 | 具体应用 |
|------|----------|
| **金融** | 股票价格预测、汇率预测、风险管理 |
| **气象** | 天气预报、空气质量预测 |
| **能源** | 电力负荷预测、太阳能发电预测 |
| **工业** | 设备故障预测、供应链需求预测 |
| **医疗** | 疾病传播预测、患者监护预警 |

### 1.2 核心方法

**数据预处理**：
```
原始数据 → 归一化 → 时间窗口划分 → 序列样本
```

**时间窗口划分**：
- 窗口大小T：决定用多少历史数据预测
- 预测步长k：决定预测多长时间的序列

```
输入: [x_{t-T}, x_{t-T+1}, ..., x_t]
输出: [x_{t+1}, x_{t+2}, ..., x_{t+k}]
```

### 1.3 模型架构选择

根据2025年最新研究(Bengio团队)：

| 模型 | 适用场景 | 优势 |
|------|----------|------|
| **LSTM** | 长期依赖、复杂模式 | 记忆能力强 |
| **GRU** | 实时性要求高 | 计算效率高 |
| **LSTM-GRU混合** | 综合任务 | 性能稳定 |
| **AttentionGRU** | 金融预测 | 结合注意力优势 |

### 1.4 实战代码框架

```python
import torch
import torch.nn as nn

class TimeSeriesPredictor(nn.Module):
    def __init__(self, input_dim, hidden_dim, num_layers, output_dim):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_dim, output_dim)
    
    def forward(self, x):
        # x: (batch, seq_len, input_dim)
        lstm_out, _ = self.lstm(x)
        # 取最后一个时间步
        prediction = self.fc(lstm_out[:, -1, :])
        return prediction
```

## 2. 文本生成

### 2.1 应用场景

- 文学创作辅助（写诗、小说）
- 代码补全（GitHub Copilot的原型）
- 对话系统回复生成
- 数据增强（生成训练样本）

### 2.2 核心技术

**字符级 vs 词级**：

| 类型 | 词汇表大小 | 优点 | 缺点 |
|------|------------|------|------|
| **字符级** | ~100 | 可处理任意单词 | 序列长，训练慢 |
| **词级** | ~30000+ | 语义丰富 | OOV问题 |

**采样策略**：
```python
def sample_with_temperature(logits, temperature=1.0):
    """温度采样控制生成多样性"""
    logits = logits / temperature
    probs = torch.softmax(logits, dim=-1)
    return torch.multinomial(probs, 1)

# temperature > 1: 更随机、多样
# temperature < 1: 更确定、保守
```

### 2.3 Shakespeare文本生成示例

使用LSTM生成莎士比亚风格的文本：

```
输入: "To be, or not to be,"
输出: "To be, or not to be, that is the question:
       Whether 'tis nobler in the mind to suffer"
```

**关键参数**：
- Sequence length: 100-200字符
- Hidden dim: 256-512
- Temperature: 0.8-1.2

## 3. 机器翻译

### 3.1 Seq2Seq经典架构

```
源语言: "I love you"
         ↓ [Encoder LSTM]
    上下文向量
         ↓ [Decoder LSTM]
目标语言: "我 爱 你"
```

### 3.2 加入注意力机制

注意力机制让Decoder在生成每个词时"关注"源句子的不同部分：

```python
class AttentionSeq2Seq(nn.Module):
    def __init__(self, encoder, decoder):
        self.encoder = encoder
        self.decoder = decoder
    
    def forward(self, src, tgt, teacher_forcing_ratio=0.5):
        # Encoder
        encoder_outputs, hidden = self.encoder(src)
        
        # Decoder with attention
        outputs = []
        current_token = tgt[:, 0:1]  # <sos>
        
        for t in range(tgt.size(1)):
            output, hidden, _ = self.decoder(current_token, hidden, encoder_outputs)
            outputs.append(output)
            
            # Teacher forcing
            if random.random() < teacher_forcing_ratio:
                current_token = tgt[:, t:t+1]
            else:
                current_token = output.argmax(dim=-1)
        
        return torch.cat(outputs, dim=1)
```

### 3.3 Transformer的崛起

虽然Seq2Seq+LSTM奠定了基础，但Transformer凭借其并行性和更好的长距离建模能力，已成为机器翻译的主流。不过，理解Seq2Seq是理解Transformer的必要前提。

## 4. 情感分析

### 4.1 任务定义

判断文本的情感倾向：
- 二分类：正面/负面
- 三分类：正面/中性/负面
- 多分类：1-5星评论

### 4.2 应用案例

| 场景 | 输入 | 输出 |
|------|------|------|
| 电商评论 | "这个产品太差了，完全不值得购买" | 负面 |
| 社交媒体 | "今天天气真不错，心情很好" | 正面 |
| 客服对话 | "请问你们的产品有什么功能？" | 中性 |

### 4.3 进阶技术

**层次化方法**：
```
句子1 → BiLSTM → hidden
句子2 → BiLSTM → hidden
句子3 → BiLSTM → hidden
    ↓
句子级别注意力 → 文档表示 → 分类
```

**预训练模型+微调**：
```
BERT/RoBERTa 编码 → LSTM层 → 分类头
```

## 5. 语音识别

### 5.1 基本流程

```
音频 → 特征提取(Mel频谱) → 编码器(LSTM/Transformer) → 解码器 → 文本
```

### 5.2 声学特征

常用特征：MFCCs、Filter Bank、Mel频谱

```
音频信号 → 分帧(25ms窗口) → 频域转换 → Mel滤波器 → 特征向量
```

### 5.3 CTC loss

Connectionist Temporal Classification (CTC) 解决了输入输出长度不对齐的问题：

```python
class SpeechRecognizer(nn.Module):
    def __init__(self, input_dim, hidden_dim, vocab_size):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, num_layers=5, 
                            bidirectional=True, batch_first=True)
        self.fc = nn.Linear(hidden_dim * 2, vocab_size)
        self.ctc_loss = nn.CTCLoss(blank=0, reduction='mean')
    
    def forward(self, x, target, input_lengths, target_lengths):
        # x: (batch, time, features)
        logits = self.fc(self.lstm(x)[0])
        log_probs = torch.log_softmax(logits, dim=-1)
        
        loss = self.ctc_loss(log_probs.transpose(0, 1), 
                            target, 
                            input_lengths, 
                            target_lengths)
        return loss
```

### 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 梯度爆炸 | 序列太长 | 梯度裁剪、层归一化 |
| 训练慢 | GPU未使用、batch太小 | 检查CUDA、小batch先跑通 |
| 过拟合 | 数据太少、模型太大 | 数据增强、正则化 |
| 预测全0 | 学习率太大 | 降低学习率、检查loss |
