# 个性化执行配置单 ## 一、配置概述 本配置单基于用户的学习目标、偏好和要求设计，明确了AI学习计划的执行框架、内容形态、验收机制和动态更新规则。 **用户画像**： - **当前基础**：Python语法基础，AI领域零经验 - **学习目标**：3个月内掌握机器学习/深度学习基础+AI工程化，达到大厂AI岗位入职水平 - **时间投入**：每周8-12小时系统学习 - **方向偏好**：机器学习/深度学习基础 + AI工程化 - **技术关注**：MCP协议、Agent架构、RAG、LLM等前沿技术 - **学习风格**：文档阅读+实战练习结合 ## 二、内容形态设计 基于用户"文档+实战"偏好，设计了五种内容形态： ### 1. 深度文档 (心知识讲解 - **格式**：Markdown文档 - **内容**：系统性讲解核心概念、数学原理、算法思想 - **示例**：`线性代数核心概念及其在机器学习与LLM中的应用.md` - **特点**：理论深度+应用场景结合，包含"今日AI动态"简报 ### 2. 代码Notebook (战练习 - **格式**：Jupyter Notebook (.ipynb 或 Python脚本 (.py - **内容**：NumPy/Pandas/Matplotlib/PyTorch实现 - **示例**：`线性代数实战练习.py` - **特点**：可运行代码+详细注释+可视化输出 ### 3. 知识测验 (握程度检验 - **格式**：Markdown文档 - **内容**：选择题、编程题、简答题 - **示例**：`线性代数知识测验.md` - **特点**：即时反馈+答案解析+知识点回顾 ### 4. 可视化图表 (念关系展示 - **格式**：ECharts配置JSON / Matplotlib图表 - **内容**：概念关系图、数据可视化、算法流程 - **示例**：`线性代数概念关系图.json` - **特点**：直观展示复杂关系，辅助理解抽象概念 ### 5. 每日AI动态简报 (沿技术跟踪 - **格式**：集成在深度文档中的独立栏目 - **内容**：最新技术进展、行业动态、研究成果 - **示例**："今日AI动态"栏目（见深度文档第6章） - **特点**：时效性强，保持学习内容与前沿同步 ## 三、验收机制设计 ### 1. 每周验收标准 - **理论学习**：完成2-3份深度文档的学习与理解 - **代码实践**：每周至少完成1个代码实战项目 - **知识测验**：每周完成5-10道测验题，正确率80%以上 - **学习笔记**：整理核心知识点，形成个人知识体系 ### 2. 阶段性评估节点 - **第一阶段结束**（第3周）：能够手动实现线性回归和逻辑回归，掌握Scikit-learn完整流程 - **第二阶段结束**（第6周）：能够使用PyTorch实现CNN图像分类，理解Transformer架构 - **第三阶段结束**（第9周）：能够构建完整的模型服务化系统，实现Docker部署 - **第四阶段结束**（第12周）：产出2-3个高质量项目，建立完整作品集 ### 3. 定期测试机制 - **知识测验**：每周生成5-10道选择题/编程题，检验理论学习效果 - **编程挑战**：每周布置1-2个编程任务，检验代码实践能力 - **项目评审**：每个阶段结束时进行项目评审，提供改进建议 - **模拟面试**：每月进行1-2次模拟面试，提升表达和应变能力 ## 四、每周循环执行模式 ### 核心执行框架（适用于Week 1-12） **每日固定**：输出《今日执行卡片》（3个核心收获+1个疑问+1个感悟） **Day 1, 3, 5（深度输入日）**： 1. **内容生成**：执行深度搜索，生成《定制学习简报》 2. **理论学习**：阅读深度文档，理解核心概念与数学原理 3. **动态跟踪**：学习"今日AI动态"，了解前沿技术进展 **Day 2, 4, 6（实战演练日）**： 1. **代码实践**：完成Notebook中的编程练习 2. **可视化**：运行代码生成图表，直观理解概念 3. **项目推进**：阶段性推进端到端项目开发 **Day 7（复盘评估日）**： 1. **周度复盘**：整理《本周执行卡片》，生成《周进展脑图》 2. **知识测验**：完成每周测验，检验学习效果 3. **计划调整**：基于进展调整下周学习重点 ## 五、动态更新规则 ### 1. 每日动态跟踪 - **触发条件**：每日深度输入日自动执行 - **执行内容**：搜索最新AI技术动态，生成"今日AI动态"简报 - **更新频率**：每日更新，确保内容时效性 - **集成方式**：作为深度文档的独立章节 ### 2. 每周计划调整 - **触发条件**：每周日复盘完成后 - **调整依据**： - 本周学习进展与测验结果 - 最新技术趋势与发展方向 - 用户反馈与难点识别 - **调整范围**： - 下周学习重点与难度 - 实践项目复杂度 - 前沿技术关注方向 ### 3. 技术栈动态纳入 - **纳入标准**： - 大厂AI岗位实际需求变化 - 行业主流技术趋势 - 开源社区活跃度与技术成熟度 - **更新机制**： - 每月技术栈评审与更新 - 基于实际应用场景的优先级调整 - 保持学习内容与实际需求的匹配度 ## 六、文件目录规范 ### 1. 核心交付目录 (`outputs/` ``` outputs/ ├── 学习规划/ # 学习路径与计划文档 │ └── AI学习路径规划书.md ├── 试学内容/ # 试学阶段内容（当前） │ ├── 线性代数核心概念及其在机器学习与LLM中的应用.md │ ├── 线性代数实战练习.py │ ├── 线性代数知识测验.md │ └── 线性代数概念关系图.json ├── 阶段一/ # 第一阶段内容 │ ├── 第1周/ │ ├── 第2周/ │ └── 第3周/ ├── 阶段二/ # 第二阶段内容 ├── 阶段三/ # 第三阶段内容 └── 阶段四/ # 第四阶段内容 ``` ### 2. 过程数据目录 (`temp/` -
