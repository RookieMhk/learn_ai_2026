### 1. 请解释Python的内存管理机制，包括引用计数、垃圾回收和内存池

**答案：**

Python的内存管理主要涉及三个方面：

1. **引用计数**：
   - 每个对象都有一个引用计数器，记录有多少个引用指向它
   - 当引用计数为0时，对象被立即回收
   - 优点：实时性好，无需等待垃圾回收器
   - 缺点：无法处理循环引用

2. **垃圾回收**：
   - **标记-清除**：解决循环引用问题，定期扫描并标记可达对象，清除不可达对象
   - **分代回收**：基于对象存活时间将对象分为不同代，年轻代回收频率高，老年代回收频率低
   - **阈值触发**：当对象数量达到阈值时触发垃圾回收

3. **内存池**：
   - 小整数对象池：缓存-5到256的整数，避免频繁创建和销毁
   - 字符串池：缓存常用字符串，实现字符串驻留
   - 内存块管理：将内存分为不同大小的块，减少内存碎片

**代码示例：**
```python
import sys
import gc

# 引用计数示例
x = 10
y = x
print(sys.getrefcount(x))  # 输出：3（包括函数参数的引用）

del y
print(sys.getrefcount(x))  # 输出：2

# 垃圾回收示例
class Node:
    def __init__(self):
        self.next = None

a = Node()
b = Node()
a.next = b
b.next = a  # 循环引用

del a
del b

# 手动触发垃圾回收
gc.collect()
print(gc.garbage)  # 输出：[]
```

### 3. 深拷贝与浅拷贝的区别

**答案：**

- **浅拷贝**：
  - 只复制对象的引用，不复制对象本身
  - 嵌套对象共享引用
  - 实现方式：`copy.copy()`、切片操作 `[:]`
  - 适用场景：复制简单结构，性能开销小

- **深拷贝**：
  - 递归复制对象及其所有嵌套对象
  - 完全独立，修改不影响原对象
  - 实现方式：`copy.deepcopy()`
  - 适用场景：复制复杂的嵌套结构，需要完全隔离的场景

**代码示例：**
```python
import copy

# 浅拷贝
original = [1, 2, [3, 4]]
shallow = copy.copy(original)
shallow[2][0] = 30
print(original)  # 输出：[1, 2, [30, 4]]

# 深拷贝
original = [1, 2, [3, 4]]
deep = copy.deepcopy(original)
deep[2][0] = 30
print(original)  # 输出：[1, 2, [3, 4]]
```

### 4. 装饰器的原理和应用

**答案：**

**原理：**
- 装饰器是一个返回函数的函数
- 利用闭包特性，在不修改原函数代码的情况下增强函数功能
- `@wraps` 装饰器用于保留原函数的元数据

**应用场景：**
- 日志记录
- 性能监控
- 权限验证
- 缓存
- 事务处理

**代码示例：**
```python
import time
from functools import wraps

def timing_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} 执行时间: {end-start:.3f}s")
        return result
    return wrapper

@timing_decorator
def slow_function():
    time.sleep(1)
    return "完成"
```

### 5. 实现LRU缓存

**答案：**

LRU（Least Recently Used）缓存是一种淘汰策略，当缓存容量达到上限时，移除最久未使用的元素。

**实现方式：**
- 使用 `OrderedDict` 实现，利用其有序特性
- 新访问的元素移到末尾（最近使用）
- 容量不足时移除头部元素（最久未使用）

**代码实现：**
```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = OrderedDict()
    
    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        
        # 移动到最近使用位置
        value = self.cache.pop(key)
        self.cache[key] = value
        return value
    
    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.pop(key)
        
        self.cache[key] = value
        
        # 超过容量删除最久未使用的
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)
```

### 6. Top K问题的解决方案

**答案：**

Top K问题是找出数据中最大（或最小）的K个元素。

**解决方案：**
1. **排序法**：对所有元素排序，取前K个，时间复杂度 O(n log n)
2. **堆排序**：使用最小堆，时间复杂度 O(n log k)
3. **快速选择**：基于快速排序的分区思想，时间复杂度 O(n)

**堆排序实现：**
```python
import heapq

def find_k_largest(nums, k):
    min_heap = []
    
    for num in nums:
        if len(min_heap) < k:
            heapq.heappush(min_heap, num)
        else:
            if num > min_heap[0]:
                heapq.heapreplace(min_heap, num)
    
    return sorted(min_heap, reverse=True)
```

### 7. 动态规划的核心思想和应用

**答案：**

**核心思想：**
- 将原问题分解为子问题
- 保存子问题的解，避免重复计算
- 自底向上或自顶向下求解

**应用场景：**
- 最长递增子序列
- 背包问题
- 编辑距离
- 矩阵路径

**最长递增子序列实现：**
```python
def longest_increasing_subsequence(nums):
    if not nums:
        return 0
    
    dp = [1] * len(nums)
    
    for i in range(1, len(nums)):
        for j in range(i):
            if nums[i] > nums[j]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)
```

   - 直接使用 Pandas 内置方法和运算符

2. **内存优化**：

   - 使用 `query()` 方法进行复杂查询
   - 使用 `isin()` 进行成员检查

4. **分组聚合优化**：
   - 使用 `agg()` 一次性完成多个聚合操作
   - 避免链式操作，减少临时对象创建

**代码示例：**
```python
import pandas as pd

# 向量化操作
df['new_col'] = df['A'] + df['B']  # 高效
# df['new_col'] = df.apply(lambda x: x['A'] + x['B'], axis=1)  # 低效

df['category'] = df['category'].astype('category')

# 查询优化
result = df.query('A > 0 and B < 10')

# 分组聚合优化
result = df.groupby('category').agg({
    'value': ['sum', 'mean', 'count'],
    'price': ['max', 'min']
})
```

## 五、并发编程

### 11. 线程池的实现和应用

**答案：**

**实现方式：**
- 使用 `concurrent.futures.ThreadPoolExecutor`
- 适用于IO密集型任务
- 可以控制并发数量，避免线程过多

**应用场景：**
- 网络请求
- 文件IO
- 数据库操作
- 并行处理任务

**代码示例：**
```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import time

def process_task(task):
    time.sleep(0.1)  # 模拟IO操作
    return f"Processed {task}"

def parallel_processing(tasks, max_workers=4):
    results = {}
    
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        future_to_task = {
            executor.submit(process_task, task): task 
            for task in tasks
        }
        
        for future in as_completed(future_to_task):
            task = future_to_task[future]
            try:
                result = future.result()
                results[task] = result
            except Exception as e:
                results[task] = f"Error: {e}"
    
    return results
```

```python
import re
from collections import defaultdict, Counter
from datetime import datetime

def parse_log_line(line):
    pattern = r'\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\] (\w+) (\w+) (.+)'
    match = re.match(pattern, line)
    
    if match:
        return {
            'timestamp': datetime.strptime(match.group(1), '%Y-%m-%d %H:%M:%S'),
            'level': match.group(2),
            'user': match.group(3),
            'message': match.group(4)
        }
    return None

def analyze_log_file(file_path):
    stats = {
        'total_lines': 0,
        'by_level': defaultdict(int),
        'by_user': Counter(),
        'by_hour': defaultdict(int),
        'errors': []
    }
    
    with open(file_path, 'r') as f:
        for line in f:
            parsed = parse_log_line(line)
            if parsed:
                stats['total_lines'] += 1
                stats['by_level'][parsed['level']] += 1
                stats['by_user'][parsed['user']] += 1
                stats['by_hour'][parsed['timestamp'].hour] += 1
                
                if parsed['level'] == 'ERROR':
                    stats['errors'].append(parsed)
    
    return stats
```
