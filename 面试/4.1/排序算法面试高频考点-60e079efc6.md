## 基础概念

### 1. 排序算法分类

**比较类排序**：
- 通过比较来决定元素间的相对次序
- 时间复杂度不能突破O(n log n)，称为比较排序的下界
- 常见算法：冒泡、选择、插入、希尔、归并、快速、堆排序

**非比较类排序**：
- 不通过比较来决定元素间的相对次序
- 可以突破O(n log n)的下界，以线性时间运行
- 常见算法：计数、桶、基数排序

### 2. 排序算法评价指标

- **时间复杂度**：最好、最坏、平均情况
- **空间复杂度**：额外内存使用情况
- **稳定性**：相等元素的相对顺序是否保持不变
- **适应性**：对已排序或部分有序数据的优化能力
- **实现复杂度**：代码实现的难易程度

## 时间复杂度分析

### 1. 时间复杂度下界

- 比较类排序的时间复杂度下界是O(n log n)
- 基于比较的排序算法，无论怎么优化，都无法突破这个下界
- 非比较类排序可以达到O(n)的时间复杂度

### 2. 各算法时间复杂度

| 算法     | 最好情况 | 最坏情况 | 平均情况 |
|----------|----------|----------|----------|
| 冒泡排序 | O(n)     | O(n²)    | O(n²)    |
| 选择排序 | O(n²)    | O(n²)    | O(n²)    |
| 插入排序 | O(n)     | O(n²)    | O(n²)    |
| 希尔排序 | O(n log n)| O(n²)   | O(n log n)| 
| 归并排序 | O(n log n)| O(n log n)| O(n log n) |
| 快速排序 | O(n log n)| O(n²)    | O(n log n) |
| 堆排序   | O(n log n)| O(n log n)| O(n log n) |
| 计数排序 | O(n + k) | O(n + k) | O(n + k) |
| 桶排序   | O(n + k) | O(n²)    | O(n + k) |
| 基数排序 | O(nk)    | O(nk)    | O(nk)    |

### 3. 时间复杂度分析技巧

- **冒泡排序**：最好情况O(n)（已排序），最坏情况O(n²)
- **快速排序**：最坏情况O(n²)（已排序或逆序），可以通过随机化 pivot 避免
- **归并排序**：始终O(n log n)，但需要O(n)的额外空间
- **堆排序**：始终O(n log n)，但不稳定

## 稳定性问题

### 1. 稳定性定义

- 稳定排序：相等元素的相对顺序在排序后保持不变
- 不稳定排序：相等元素的相对顺序可能改变

### 2. 各算法稳定性

**稳定排序**：
- 冒泡排序
- 插入排序
- 归并排序
- 计数排序
- 桶排序
- 基数排序

**不稳定排序**：
- 选择排序
- 希尔排序
- 快速排序
- 堆排序

### 3. 稳定性的重要性

- **多关键字排序**：例如先按部门排序，再按工资排序
- **对象排序**：保持对象的原始顺序
- **数据关联**：排序时保持相关数据的对应关系

## 算法比较与选择

### 1. 算法选择依据

| 因素           | 考虑点                                                                 |
|----------------|------------------------------------------------------------------------|
| **数据规模**   | 小规模用简单算法，大规模用高效算法                                     |
| **数据分布**   | 已排序数据用插入排序，随机分布用快速排序，均匀分布用桶排序             |
| **稳定性要求** | 需要稳定排序时选择归并、计数、基数等算法                               |
| **内存限制**   | 内存有限时选择原地排序算法（如堆排序、快速排序）                       |
| **实现复杂度** | 优先选择易于实现和维护的算法                                           |

### 2. 常见场景推荐

| 场景               | 推荐算法               |
|--------------------|------------------------|
| 通用场景           | 快速排序               |
| 稳定排序           | 归并排序、基数排序     |
| 内存受限           | 堆排序、希尔排序       |
| 整数排序           | 计数排序、基数排序     |
| 浮点数排序         | 桶排序、快速排序       |
| 小规模数据         | 插入排序、冒泡排序     |
| 部分有序数据       | 插入排序、冒泡排序     |

## 常见面试题

### 1. 基础概念题

**问题**：什么是稳定排序？为什么稳定性很重要？
**回答**：
- 稳定排序是指相等元素的相对顺序在排序后保持不变
- 稳定性在多关键字排序和对象排序中非常重要
- 例如，先按部门排序，再按工资排序时需要保持部门内的相对顺序

**问题**：比较类排序的时间复杂度下界为什么是O(n log n)？
**回答**：
- 基于比较的排序可以看作是一棵决策树
- 每个叶子节点代表一种可能的排序结果
- 有n!种可能的排序结果，决策树的高度至少是log2(n!) ≈ n log n

### 2. 算法实现题

**问题**：实现快速排序，并优化最坏情况
**回答**：
```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    # 随机选择pivot避免最坏情况
    pivot = random.choice(arr)
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)
```

**问题**：实现归并排序，并分析空间复杂度
**回答**：
```python
def merge_sort(arr):
    if len(arr) > 1:
        mid = len(arr) // 2
        left = arr[:mid]
        right = arr[mid:]
        merge_sort(left)
        merge_sort(right)
        
        i = j = k = 0
        while i < len(left) and j < len(right):
            if left[i] < right[j]:
                arr[k] = left[i]
                i += 1
            else:
                arr[k] = right[j]
                j += 1
            k += 1
        
        while i < len(left):
            arr[k] = left[i]
            i += 1
            k += 1
        
        while j < len(right):
            arr[k] = right[j]
            j += 1
            k += 1
    return arr
```
- 空间复杂度O(n)，需要额外的存储空间来合并子数组

### 3. 算法优化题

**问题**：如何优化冒泡排序？
**回答**：
- 添加标志位，若某轮没有交换则提前结束
- 记录最后一次交换的位置，减少比较次数
- 双向冒泡排序（鸡尾酒排序），适用于部分有序的数据

**问题**：为什么快速排序比归并排序更快？
**回答**：
- 快速排序的常数因子更小
- 快速排序是原地排序，缓存局部性更好
- 归并排序需要额外的内存空间，有内存开销

### 4. 综合应用题

**问题**：如何对海量数据进行排序？
**回答**：
- 采用外部排序，分块读取数据到内存
- 对每个块进行排序后写入磁盘
- 使用归并排序合并多个已排序的块
- 可以结合多路归并提高效率

**问题**：如何实现一个通用的排序函数？
**回答**：
- 允许用户自定义比较函数
- 处理不同的数据类型（整数、浮点数、字符串、对象）
- 考虑稳定性和性能平衡
- 实现自适应排序，根据数据特点选择合适的算法

## 代码实现技巧

### 1. 冒泡排序优化

```python
def optimized_bubble_sort(arr):
    n = len(arr)
    last_swap = n - 1
    for i in range(n):
        swapped = False
        for j in range(last_swap):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
                swapped = True
                last_swap = j
        if not swapped:
            break
    return arr
```

### 2. 快速排序优化

```python
def quick_sort_optimized(arr, low=0, high=None):
    if high is None:
        high = len(arr) - 1
    
    if low < high:
        # 三数取中法选择pivot
        mid = (low + high) // 2
        if arr[low] > arr[mid]:
            arr[low], arr[mid] = arr[mid], arr[low]
        if arr[low] > arr[high]:
            arr[low], arr[high] = arr[high], arr[low]
        if arr[mid] > arr[high]:
            arr[mid], arr[high] = arr[high], arr[mid]
        arr[mid], arr[high-1] = arr[high-1], arr[mid]
        pivot = arr[high-1]
        
        i, j = low, high-1
        while True:
            i += 1
            while arr[i] < pivot:
                i += 1
            j -= 1
            while arr[j] > pivot:
                j -= 1
            if i < j:
                arr[i], arr[j] = arr[j], arr[i]
            else:
                break
        arr[i], arr[high-1] = arr[high-1], arr[i]
        
        quick_sort_optimized(arr, low, i-1)
        quick_sort_optimized(arr, i+1, high)
    return arr
```

### 3. 归并排序迭代实现

```python
def merge_sort_iterative(arr):
    n = len(arr)
    current_size = 1
    
    while current_size < n:
        left = 0
        while left < n - current_size:
            mid = left + current_size
            right = min(left + 2*current_size - 1, n-1)
            
            # 合并两个子数组
            i = left
            j = mid
            k = 0
            temp = [0]*(right-left+1)
            
            while i < mid and j <= right:
                if arr[i] < arr[j]:
                    temp[k] = arr[i]
                    i += 1
                else:
                    temp[k] = arr[j]
                    j += 1
                k += 1
            
            while i < mid:
                temp[k] = arr[i]
                i += 1
                k += 1
            
            while j <= right:
                temp[k] = arr[j]
                j += 1
                k += 1
            
            for i in range(left, right+1):
                arr[i] = temp[i-left]
            
            left += 2*current_size
        current_size *= 2
    return arr
```

## 进阶问题

### 1. 外部排序

**问题**：如何排序无法全部放入内存的海量数据？
**思路**：
- 分治法：将数据分成多个块，分别排序后写入磁盘
- 多路归并：同时合并多个已排序的块
- 败者树：优化多路归并的比较过程
- 置换选择：生成更长的初始有序块

### 2. 并行排序

**问题**：如何利用多处理器加速排序？
**思路**：
- 并行快速排序：将数据分成多个部分并行处理
- 并行归并排序：并行划分和合并
- 基数排序的并行化
- 利用MapReduce框架进行分布式排序

### 3. 自适应排序

**问题**：如何根据数据特点自动选择最优算法？
**思路**：
- 检测数据的有序程度
- 小规模数据用插入排序
- 大规模数据用快速排序或归并排序
- 已排序数据用插入排序

### 2. 常见问题回答模板

**问题**：请比较快速排序和归并排序
**回答**：
> 快速排序和归并排序都是分治算法，时间复杂度都是O(n log n)，但有以下区别：
> 
> 1. **排序方式**：快速排序是原地排序，归并排序需要额外内存
> 2. **稳定性**：快速排序不稳定，归并排序稳定
> 3. **性能**：快速排序常数因子更小，实际运行更快
> 4. **最坏情况**：快速排序最坏情况O(n²)，归并排序始终O(n log n)
> 5. **适用场景**：快速排序适用于通用场景，归并排序适用于需要稳定排序或外部排序的场景

- 可以提供递归和迭代两种实现方式

- 说明时间复杂度的推导过程
